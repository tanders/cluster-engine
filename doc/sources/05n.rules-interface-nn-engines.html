shift-start-before-timepoint-in-list   start-point list offset  [Function]

The objective is to move offset-steps before the start point in the list. Starttime does not have to exist in the list. If the offset is larger is to before the start point, the start point will be given. If gracenotes are included, they will be considered before start-point (i.e. this is probably a not desured side effect for rhythm, but it workd for metric structures such as beats or metric grid)
